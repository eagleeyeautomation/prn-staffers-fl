# Eagle Eye Command Center Backlog

## Backlog Rules

- Version 1.0 uses mock data only.
- Do not connect live APIs until approved.
- Keep Eagle Eye Command Center branding.
- Keep reusable components and centralized data providers.
- Prioritize features that help executives understand company health quickly.
- Run `pnpm lint` and `pnpm build` after completed work.
- Commit each completed feature.

## Priority Levels

- P0: Required for Version 1.0 foundation.
- P1: High-value feature for the next release.
- P2: Important enhancement after core workflows are stable.
- P3: Future opportunity.
